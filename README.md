# SwiftTouch

Sitio web de presencia digital para SwiftTouch, una empresa tecnológica hondureña
orientada a la innovación, la infraestructura digital y soluciones empresariales.

La aplicación está desarrollada con **React**, **Tailwind CSS** y **Framer Motion**
para una experiencia moderna, minimalista y completamente responsive.

Está pensada como una landing page corporativa que transmite profesionalismo,
seguridad y visión a largo plazo sin ofrecer servicios específicos o precios.

El proyecto fue inicialmente creado usando Create React App, y la configuración
se ha adaptado para Tailwind CSS.

## Available Scripts

In the project directory, you can run:

### `npm start`

Arranca el servidor de desarrollo en `http://localhost:3000`.


Runs the app in the development mode.\
Open [http://localhost:3000](http://localhost:3000) to view it in your browser.

The page will reload when you make changes.\
You may also see any lint errors in the console.

### `npm test`

Launches the test runner in the interactive watch mode.\
See the section about [running tests](https://facebook.github.io/create-react-app/docs/running-tests) for more information.

### `npm run build`

Genera la versión optimizada para producción en la carpeta `build`.
Los archivos se minifican y se incorporan hashes para cache busting. El resultado
puede ser desplegado en cualquier servicio de hosting estático como **Vercel**,
**Netlify** o un bucket de **AWS S3** (con CloudFront).

La aplicación no requiere configuración de servidor adicional: basta con subir
el contenido de `build`.

Ejemplos de despliegue:

- **Vercel**: conectar el repositorio y Vercel detectará automáticamente el
  proyecto Create React App. El comando de build ya está establecido en
  `npm run build`.
- **AWS Amplify / S3+CloudFront**: subir la carpeta `build` a un bucket S3 y
  configurar CloudFront para servirla. Ajustar variables de entorno si lo
  deseas.


### `npm run eject`

**Note: this is a one-way operation. Once you `eject`, you can't go back!**

If you aren't satisfied with the build tool and configuration choices, you can `eject` at any time. This command will remove the single build dependency from your project.

Instead, it will copy all the configuration files and the transitive dependencies (webpack, Babel, ESLint, etc) right into your project so you have full control over them. All of the commands except `eject` will still work, but they will point to the copied scripts so you can tweak them. At this point you're on your own.

You don't have to ever use `eject`. The curated feature set is suitable for small and middle deployments, and you shouldn't feel obligated to use this feature. However we understand that this tool wouldn't be useful if you couldn't customize it when you are ready for it.
